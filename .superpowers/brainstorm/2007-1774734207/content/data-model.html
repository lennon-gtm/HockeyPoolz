Section 2 — Data Model

Every table and how they connect. Supabase PostgreSQL.

USERS
id uuid PK
email text unique
display_name text
avatar_url text
provider google | apple
is_platform_admin bool
created_at timestampz
LEAGUES
id uuid PK
commissioner_id → users
name text
invite_code text unique
status setup | draft | active | complete
max_teams int
created_at timestampz
LEAGUE_MEMBERS
id uuid PK
league_id → leagues
user_id → users
team_name text
draft_position int
total_score decimal (cached)
joined_at timestampz
↕ league_id links scoring, draft, and members
SCORING_SETTINGS
id uuid PK
league_id → leagues (1:1)
goals decimal weight
assists decimal weight
plus_minus decimal weight
pim decimal weight
shots decimal weight
goalie_wins decimal weight
goalie_saves decimal weight
shutouts decimal weight
DRAFTS
id uuid PK
league_id → leagues (1:1)
status pending | active | complete
current_pick_number int
pick_time_limit_secs int
started_at timestampz
DRAFT_PICKS
id uuid PK
draft_id → drafts
league_member_id → league_members
player_id → nhl_players
round int
pick_number int
picked_at timestampz
↕ StatsService writes to nhl_players + player_game_stats
NHL_TEAMS (StatsService)
id text (e.g. "TOR")
name text
logo_url text
conference text
eliminated_at timestampz | null
NHL_PLAYERS (StatsService)
id int (NHL player ID)
team_id → nhl_teams
name text
position C|LW|RW|D|G
headshot_url text
PLAYER_GAME_STATS (StatsService)
id uuid PK
player_id → nhl_players
game_id text (NHL game ID)
game_date date
goals, assists, plus_minus
pim, shots
wins, saves, shutouts (G)

■ Primary key
